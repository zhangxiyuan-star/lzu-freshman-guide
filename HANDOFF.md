## 一、我们在做什么任务

### 项目目标
为兰州大学新生制作一个**公开可访问的静态网页入学手册**，把报到前后最容易卡住的事情整理在一个网页里，替代分散在微信群、公众号、贴吧里的零散信息。

### 核心价值
- **信息集中**：宿舍、食堂、地图、校园卡、军训、转专业、校历、快递、通勤等高频问题一站式入口
- **稳定维护**：采用"稳定说明 + 当年官方通知链接"的方式，避免内容过时
- **公开访问**：纯静态，可部署到 GitHub Pages / Vercel / 学校服务器，新生无需登录即可查看

### 目标用户
- 兰州大学本科新生（主要是榆中校区）
- 部分研究生（城关校区）
- 送新生的家长

## 五、踩过的坑（绝对不要踩）

### 1. ❌ 不要重复定义函数
曾在 `app.js` 里把 `initDisclaimer()` 写了两次，浏览器虽不报错（后者覆盖前者），但维护时改一处忘另一处会出 bug。**加新功能前先扫一遍文件，确认没有重复代码块。**

### 2. ❌ 不要用招生海报当首屏背景图
`campus-hero-photo.jpg`、`campus-scene.jpg`、`campus-a.jpeg` 实际上都是招生宣传海报（带大字"26兰大本科招""8大改革亮点"），当背景会被深色遮罩盖住但依然很丑。**首屏用 `campus-hero.svg`（插画风格），等有授权的真实校园照片再替换。**

### 3. ❌ 不要直接编辑压缩后的 CSS
`styles.css` 是全部压在两行里的，还带 BOM 头。直接改容易改错选择器。**少量样式修改可以通过 JS 动态注入（参考 app.js 末尾的 addStyle 写法），大量修改先格式化CSS。**

### 4. ❌ 不要写死具体年份/日期
不要在正文里写"2026年军训时间是X月X日"，下一年就过时了。正确做法：写通用说明 + 放当年官方通知链接。唯一要更新的是 `verifiedDate`（核验日期）。

### 5. ❌ 不要放未经证实的信息
- 校园墙、互助群这类学生自发运营的入口，要确认还在维护再放
- 个人手机号、辅导员电话**绝对不要未经允许公开**
- 所有信息标注"以官方通知为准"，免责声明不能少

### 6. ❌ 不要加构建工具/框架
这个项目的优势就是**简单**——一个 html、一个 css、一个 js，双击就能打开。不要引入 React/Vite/Webpack 之类的，部署和维护成本会飙升。纯静态足够了。

### 7. ❌ 不要忘记本地测试后再提交/push
- 改完代码一定要在浏览器里打开试试：搜索能用吗？筛选能用吗？弹窗正常吗？
- 特别注意：浏览器可能缓存旧图片/旧JS，用 `?v=2` 之类的参数强制刷新
- 所有外链点一遍，确认不是 404

### 8. ❌ 文件路径用相对路径，不要用绝对路径
因为可能部署到 GitHub Pages 的子路径（/lzu-freshman-guide/），也可能部署到 Vercel 根域名。用 `assets/xxx.jpg` 这种相对路径，两边都能用。

### 9. ❌ 图片不要太大，注意图片内容
- 首屏图片控制在 500KB 以内，手机加载慢
- **用图片前先看一眼图片内容！** 不要看文件名就假设它是校园照片，这次就踩了这个坑
