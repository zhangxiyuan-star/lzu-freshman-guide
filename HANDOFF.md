## 一、我们在做什么任务

### 项目目标
为兰州大学新生制作一个**公开可访问的静态网页入学手册**，把报到前后最容易卡住的事情整理在一个网页里，替代分散在微信群、公众号、贴吧里的零散信息。

### 核心价值
- **信息集中**：宿舍、食堂、地图、校园卡、军训、转专业、校历、快递、通勤等高频问题一站式入口，新增留言墙与词云互动、目录快速导航
- **稳定维护**：采用"稳定说明 + 当年官方通知链接"的方式，避免内容过时
- **公开访问**：纯静态，可部署到 GitHub Pages / Vercel / 学校服务器，新生无需登录即可查看

### 目标用户
- 兰州大学本科新生（主要是榆中校区）
- 部分研究生（城关校区）
- 送新生的家长

## 五、踩过的坑（绝对不要踩）

### 1. ❌ 不要重复定义函数
曾在 `app.js` 里把 `initDisclaimer()` 写了两次，浏览器虽不报错（后者覆盖前者），但维护时改一处忘另一处会出 bug。**加新功能前先扫一遍文件，确认没有重复代码块。**

### 2. ❌ 不要用招生海报当背景图
`campus-hero-photo.jpg`、`campus-scene.jpg`、`campus-a.jpeg` 实际上都是招生宣传海报（带大字宣传语），当背景会被深色遮罩盖住但依然很丑。**首屏现在用 hero-campus.jpg（积石堂实拍）。用图片前必须查看内容，不能仅凭文件名判断。**

### 3. ✅ 压缩 CSS 问题已解决
`styles.css` 曾经是全部压在两行里的，还带 BOM 头。**现已格式化为 2002 行完整版本，可直接编辑。** 新教训：**接手项目时先检查 CSS/JS 是否为压缩状态，如果是先格式化再改，不要在压缩状态下硬改。**

### 4. ❌ 不要写死具体年份/日期
不要在正文里写"2026年军训时间是X月X日"，下一年就过时了。正确做法：写通用说明 + 放当年官方通知链接。

### 5. ❌ 不要放未经证实的信息
- 校园墙、互助群这类学生自发运营的入口，要确认还在维护再放
- 个人手机号、辅导员电话**绝对不要未经允许公开**
- 所有信息标注"以官方通知为准"，免责声明不能少

### 6. ❌ 不要加构建工具/框架
这个项目的优势就是**简单**——一个 html、一个 css、一个 js，双击就能打开。不要引入 React/Vite/Webpack 之类的，部署和维护成本会飙升。纯静态足够了。

### 7. ❌ 不要忘记本地测试后再提交/push
- 改完代码一定要在浏览器里打开试试：搜索能用吗？筛选能用吗？弹窗正常吗？
- 特别注意：浏览器可能缓存旧图片/旧JS，用 `?v=2` 之类的参数强制刷新
- 所有外链点一遍，确认不是 404

### 8. ❌ 文件路径用相对路径，不要用绝对路径
因为可能部署到 GitHub Pages 的子路径（/lzu-freshman-guide/），也可能部署到 Vercel 根域名。用 `assets/xxx.jpg` 这种相对路径，两边都能用。

### 9. ❌ 图片不要太大，注意图片内容
- 首屏图片控制在 500KB 以内，手机加载慢
- **用图片前先看一眼图片内容！** 不要看文件名就假设它是校园照片，之前就踩过这个坑

### 10. ❌ 词云数据上限要提前设计
留言墙最多 30 条词是硬限制，超出后新留言如何处理（替换最旧的？拒绝提交？提示已满？）需要明确策略。当前实现是保留最新 30 条。

### 11. ❌ UGC 内容要有审核意识
留言墙允许用户提交文字，虽然只是前端本地存储，但上线后如果接入后端，**必须有内容审核机制**，防止不当内容出现在词云中。

### 12. ❌ Hero 背景图要兼顾桌面和移动端
实拍照片（如积石堂）在桌面端效果好，但在移动端可能主体被裁切。需要测试不同屏幕尺寸下的显示效果，必要时用 `object-position` 调整或准备移动端专用图。

### 13. ❌ 内联 SVG 图标注意可维护性
高频入口用了内联 SVG，图标多了以后 HTML 文件会膨胀。如果后续图标继续增加，可以考虑转成 SVG sprite 或图标字体。

## 八、部署指南（给新人）

### 本地预览
```bash
cd f:\新生网页
python -m http.server 8080
# 浏览器访问 http://localhost:8080
# 强制刷新防缓存：http://localhost:8080/?v=2
```

### 部署到 GitHub Pages（已配置）
1. `git add . && git commit -m "xxx" && git push`
2. 等 1-2 分钟，访问 https://zhangxiyuan-star.github.io/lzu-freshman-guide/
3. GitHub Pages 国内可直接访问

### 部署到 Vercel（已导入）
1. Vercel 已连接 GitHub 仓库，每次 `git push` 自动部署
2. 项目地址：https://vercel.com/zhang-xiyuan/lzu-freshman-guide
3. Framework Preset 必须选 **Other**（纯静态，无构建步骤）
4. `.vercel.app` 域名国内可能需要科学上网

### 部署后检查清单
- 搜索功能正常
- 筛选功能正常
- 目录导航弹窗正常
- 留言墙提交 + 词云更新正常
- 回到顶部按钮正常
- 所有外链可点击跳转正常
- 图片全部加载成功
